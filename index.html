Seeing is Believing

Your tool should create a mental model at a glance.

  • Visibility of System Status
  • Match between System and the Real World
  • Recognition rather than Recall
Fast feedback
Notation, format should be familiar
Reduce cognitive load
Shoot for the Sky (not your Foot)

Powerful tools are great -- if they don't destroy stuff.

  • User Freedom
  • Flexibility and Efficiency
  • Error Prevention
  • Error Discovery and Recovery
Users will and should make mistakes
Errors are best prevented, but if not, easy to recover from
The Best Kind of Rut

Offer exactly one clear way to perform tasks.

  • Consistency
  • Aesthetics and Minimalism
  • Help and Documentation
Words/Actions should always mean the same thing
Docs and Aesthetics should both have the goldilocks amount
Docs should be easy to search, and offer actionable tutorial
